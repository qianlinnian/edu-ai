- 测试类型：功能测试 + UI一致性测试 + 接口联调测试（黑盒为主）

  - 浏览器：Chrome（最新稳定版）

## 2. 缺陷列表

### BUG-001 注册页面无法完成注册（高）
- 问题标题：注册功能提交后无法完成有效注册
- 接口地址：`POST /api/v1/auth/register`
- 环境：本地开发环境（前端 + 后端）
- 复现步骤：
  1. 打开登录页并切换到“注册”
  2. 输入用户名、邮箱、姓名、密码、角色
  3. 点击“注册”
  4. 观察前端提示与后续登录结果
- 预期结果：
  - 返回注册成功
  - 账号可立即用于登录
- 实际响应：
  - 前端无法完成预期注册流程（表现为“注册不可用”）
  - 需要后端进一步排查接口返回与数据库写入链路

---

### BUG-002 登录/注册输入框文字输入后背景变白（中）
- 问题标题：登录页与注册页输入框视觉风格不一致
- 接口地址：`N/A（UI问题）`
- 环境：登录页 `http://localhost:5173/login`
- 复现步骤：
  1. 进入登录页
  2. 在登录或注册表单输入用户名/密码等内容
  3. 观察输入框底色变化
- 预期结果：
  - 输入前后均保持深色主题样式（与页面统一）
- 实际响应：
  - 输入后部分输入框底色变为白色，和整体深色背景冲突

---

### BUG-003 学生端-练习中心选项文字与背景颜色冲突（中）
- 问题标题：练习中心选项文本可读性差（白底白字）
- 接口地址：`N/A（UI问题）`
- 环境：`http://localhost:5173/exercises`
- 复现步骤：
  1. 进入练习中心页面
  2. 查看题目选项区域
  3. 在选中/未选中状态下观察选项文本颜色
- 预期结果：
  - 选项文字始终清晰可读，与背景保持足够对比度
- 实际响应：
  - 选项文字与背景颜色接近，出现阅读困难

---

### BUG-004 练习中心进度条计算不准确（中）
- 问题标题：第一题时进度显示为0%
- 接口地址：`N/A（前端逻辑问题）`
- 环境：`http://localhost:5173/exercises`
- 复现步骤：
  1. 进入练习中心
  2. 观察 Q1/总题数 对应的进度条百分比
- 预期结果：
  - 第一题应展示已进入第1题进度（如 20%）
- 实际响应：
  - 第一题显示 0%，用户感知进度偏差

---

### BUG-005 注册失败时前端未展示后端错误明细（低）
- 问题标题：注册失败错误提示过于笼统，不利于定位
- 接口地址：`POST /api/v1/auth/register`
- 环境：登录页注册表单
- 复现步骤：
  1. 在注册页输入已存在用户名或邮箱
  2. 提交注册
  3. 观察提示信息
- 预期结果：
  - 前端展示后端返回的明确原因（如“用户名已存在”）
- 实际响应：
  - 统一提示“注册失败，请检查信息后重试”，缺乏可操作信息

## 4. 测试结论与建议
- 当前版本可用于内部联调演示，但不建议直接面向正式用户发布。
- 建议优先修复注册链路可用性（BUG-001），随后修复UI一致性与可读性问题（BUG-002、BUG-003）。
- 建议补充前端错误处理策略：优先展示后端可读错误信息，降低排障成本。
